# Voyant Documentation: Workflows & Pure Execution (`apps/workflows`, `apps/analysis`, `apps/scraper`)

## 1. Overview
This module combination manages the execution of complex, asynchronous tasks within Voyant. It is built strictly on the "Agent-Tool Architecture," treating the backend as a pure execution pipeline (no autonomous LLMs) that performs mechanical work dispatched via Temporal.

## 2. File-by-File Breakdown

### `apps/workflows/api.py`
The orchestration hub for data ingestion, profiling, and quality checks.
*   **Trigger Endpoints (`/ingest`, `/profile`, `/quality`):**
    *   Creates a `Job` record in the database.
    *   Binds to the Soma Integration layer (`create_task_for_job`) to sync state with the external agent.
    *   Dispatches Temporal workflows (`IngestDataWorkflow`, `ProfileWorkflow`, `QualityWorkflow`) asynchronously to `temporal_task_queue`.
*   **Job Management (`/jobs`, `/jobs/{job_id}/cancel`):** Lists jobs and acts as the cancellation gateway, stopping running workflows in Temporal.
*   **Artifacts (`/artifacts/{job_id}`, `/artifacts/../download`):** Integrates exclusively with MinIO to retrieve or stream resulting artifacts (`JSON`, `CSV`, etc.) generated by the jobs.
*   **Presets & KPI Templates (`/presets`, `/presets/kpi-templates`):** Allows execution of predefined, templated jobs (like `quality.data_profiling`) or rendering of templated SQL aggregations (`render_kpi_template`).

### `apps/analysis/api.py`
Executes deep statistical profiling and quality evaluation over data sources.
*   **Trigger (`POST /analyze`):** Extremely robust gateway.
    *   Takes `source_id`, `table`, physical bounds (`sample_size`), and optional `analyzers` or `KPIQuery` objects.
    *   Enforces Data Sovereignty by explicitly running `validate_table_access(tenant_id, table)` (catching `NamespaceViolationError`).
    *   Integrates with the Soma agent (`remember_summary`).
    *   Dispatches the `AnalyzeWorkflow` to Temporal.

### `apps/scraper/api.py`
The mechanical web interaction and media extraction service. It is explicitly LLM-free.
*   **Workflow Jobs (`/start`):** Starts a Temporal `ScrapeWorkflow` to visit URLs. Employs critical **SSRF validation** (`validate_urls`) before hitting the web. Accepts custom `engine` parameters (Playwright/Scrapy/httpx).
*   **Synchronous Hooks (`/fetch`, `/ocr`, `/parse_pdf`, `/transcribe`):** Single-shot execution tools that leverage `ScrapeActivities`.
    *   `/fetch`: Connects via Playwright/httpx and respects directives like `scroll`, `wait_for`, and resource blocking.
    *   `/ocr`: Uses Tesseract/Vision APIs without NLP.
    *   `/transcribe`: Uses Whisper via `ScrapeActivities` (if `scraper_enable_transcribe` is True in settings).
*   **Synchronous Parsing (`/extract`):** Uses `lxml` and `CSSSelector` to evaluate deterministic XPath/CSS queries against raw HTML provided by an agent. No text generation occurs.

## 3. Core Principles Reflected
*   **Agent-Agnostic Execution:** The scraper exposes raw capabilities (`/start`, `/fetch`) without deciding *what* to scrape. The agent decides.
*   **Temporal First:** The API layers are kept thin and fast. Any operation taking longer than a second is dropped into `Job` states and dispatched to `temporalio`.
*   **Data Security:** Consistent use of `get_tenant_id()`, `validate_table_access()`, and `apply_policy()` to enforce compliance bounds at the router level.
